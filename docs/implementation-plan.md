# Implementation Plan — Vertical Slices

Each row is a vertical slice: server + client + DB end-to-end, demonstrable in the running app. Slices are ordered to unblock the next; "Depends on" makes the order auditable.

`todo` · `in-progress` · `blocked` · `done`

## Sequencing notes

- **Slice 0 unblocks everything** — no other slice can land without `ChatDbContext` and migration-on-startup wiring.
- **Slices 1–6 form the "two-user DM + presence" milestone** — earliest moment the app feels real. Good demo checkpoint.
- **Slices 7–10 form the "rooms + unread" milestone** — feature-complete chat for a small group.
- **Slices 11–14 add the social/admin surface** — attachments, invites, moderation, blocks.
- **Slices 15–16 are cross-cutting polish** that touch many earlier slices; doing them earlier risks rework, doing them later risks a late-stage scramble. Suggested as written.
- **Slice 17 is the verification gate** before declaring MVP complete; unit tests for permissions should land *with* their owning slice rather than waiting until the end.
